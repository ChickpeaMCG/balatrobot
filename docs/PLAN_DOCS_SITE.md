# Documentation Site Plan: MkDocs + GitHub Pages

## Goal

A public-facing website that chronicles the evolution of Balatrobot across phases — what each version could do, what decisions were made, what bugs were found — and eventually displays live run analytics as charts. The site should update automatically when code is pushed to `main`.

---

## Why MkDocs + Material + GitHub Pages

| Concern | Answer |
|---|---|
| Hosting cost | Free — GitHub Pages serves static files |
| Deployment complexity | One command (`mkdocs gh-deploy`) or one GitHub Actions workflow |
| Python ecosystem fit | MkDocs is a Python tool; configured in `pyproject.toml`; same `pip install` as the rest of the project |
| Writing experience | Pure Markdown, same as existing `docs/PLAN*.md` files |
| Chart support | Material theme supports custom JS blocks; Chart.js embeds cleanly |
| Live data | A pre-build script exports `run_history.json` → `docs/data/runs.json`; Chart.js reads it client-side |

The **Material for MkDocs** theme (`mkdocs-material`) is the standard choice. It adds search, dark/light mode, responsive layout, and admonition blocks without any extra configuration.

## Dependencies

Add to `pyproject.toml` under `[project.optional-dependencies]`:

```toml
[project.optional-dependencies]
docs = [
    "mkdocs>=1.6",
    "mkdocs-material>=9.5",
]
```

Install with:
```bash
pip install -e ".[docs]"
```

## Data Pipeline: `run_history.json` → Charts

A small script converts the existing run history into a static JSON file that Chart.js reads in the browser.

**`scripts/export_runs.py`** (new file):
```python
"""Export run_history.json → docs/data/runs.json for the analytics page."""
import json
from pathlib import Path

src = Path("run_history.json")
dst = Path("docs/data/runs.json")

if not src.exists():
    dst.write_text("[]")
else:
    history = json.loads(src.read_text())
    dst.parent.mkdir(parents=True, exist_ok=True)
    dst.write_text(json.dumps(history["runs"], indent=2))
```

Run before `mkdocs build`:
```bash
python scripts/export_runs.py && mkdocs build
```

**`docs/js/runs_chart.js`** (new file):
```javascript
document.addEventListener("DOMContentLoaded", async () => {
  const canvas = document.getElementById("runs-chart");
  if (!canvas) return;

  const res = await fetch("../data/runs.json");
  const runs = await res.json();

  const labels = runs.map((_, i) => `Run ${i + 1}`);
  const anteData = runs.map(r => r.ante_reached);

  new Chart(canvas, {
    type: "bar",
    data: {
      labels,
      datasets: [{
        label: "Ante Reached",
        data: anteData,
        backgroundColor: "rgba(239, 68, 68, 0.7)",
      }]
    },
    options: {
      scales: { y: { beginAtZero: true, max: 8 } },
      plugins: { legend: { display: false } }
    }
  });
});
```

## GitHub Actions: Auto-Deploy on Push

**`.github/workflows/docs.yml`** (new file):

```yaml
name: Deploy Docs

on:
  push:
    branches: [main]

permissions:
  contents: write   # needed to push to gh-pages branch

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-python@v5
        with:
          python-version: "3.11"

      - name: Install dependencies
        run: pip install -e ".[docs]"

      - name: Export run data
        run: python scripts/export_runs.py

      - name: Build and deploy
        run: mkdocs gh-deploy --force
```

This pushes the built site to the `gh-pages` branch. Enable GitHub Pages in repository Settings → Pages → Source: `gh-pages` branch, `/ (root)`.

## Implementation Steps

| Step | Task | Verify |
|---|---|---|
| 1 | `pip install mkdocs-material`, create `mkdocs.yml` at repo root | `mkdocs serve` renders a local site |
| 2 | Write `docs/index.md` — project overview | Page visible at `localhost:8000` |
| 3 | Write `docs/architecture.md` — system overview | Diagrams/tables render |
| 4 | Create `scripts/export_runs.py` | `docs/data/runs.json` generated |
| 5 | Create `docs/js/runs_chart.js` and `docs/analytics/runs.md` | Chart renders with data |
| 6 | Add `[docs]` extras to `pyproject.toml` | `pip install -e ".[docs]"` works |
| 7 | Create `.github/workflows/docs.yml` | Push to main triggers deploy |
| 8 | Enable GitHub Pages in repo settings | Site live at `<username>.github.io/balatrobot` |

Phase pages require no extra work — each phase record in `docs/superpowers/records/` is added to `mkdocs.yml` nav when the branch merges.

## What's Out of Scope For Now

- **Live data from active runs** — would require a backend; the static JSON approach is sufficient until Phase 5
- **Authentication or private pages** — site is public
- **Video embeds or GIFs** — could be added later to phase pages to show the bot playing
- **Auto-generated API reference** — the Python package isn't a public API; no `mkdocstrings` needed yet
- **Comments / feedback** — not needed for a development chronicle

---

## Future: ML Learning Curves (Phase 5 Hook)

When Phase 5 (RL) begins, `export_runs.py` can be extended to export training metrics (reward per episode, policy loss, ante distribution over time) as additional JSON files. The analytics page gains new chart sections without any structural changes to the site.

The Chart.js bar chart in `runs_chart.js` becomes a line chart showing learning curves — reward vs. episode number — with a second dataset showing ante reached distribution as training progresses.
